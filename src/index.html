How we implement the operations

  • Multiply

    We can run two DFAs in parallel on the same input via the product construction, as long as they share the same alphabet. Q1x2 = Q1 x Q2.

  • Concatenate

    The set of states of the concatenated NFA is just the (disjoint) union of the states of the two automata. The initial state of the new NFA is the initial state of the first NFA. The accepting states of the new NFA are the accepting states of the second NFA. Every transition from either original NFA is a transition in the new NFA. The only new transitions are εε-moves from each accepting state of the first NFA to the initial state of the second NFA.

  • Complement

    If DFA accepts language L, then L is accepted by DFA, a version of complement DFA in which the accepting and non-accepting states have been swapped.

  • Reverse

    Assigns to each pair of strings (x;y) what T assigns to its mirror images (x;y)

  • Intersection

    We can intersect DFA1 and DFA2 (written DFA1 ∩ DFA2) by defining the accepting states of DFA1 x DFA2 as those state pairs in which both states are final states of their DFAs.

  • Union

    We can union DFA1 and DFA2 (written DFA1 ∪ DFA2) by defining the accepting states of DFA1 x DFA2 as those state pairs in which either state is a final state of its DFA.

Upload dot file instructions

Your code need to follow the following :

 FA A = new("A.dot");
 FA B = new("B.dot");
 FA C = A int B;
 FA D = rev(A);
 FA E = A + B;
 FA F= A . B;
 FA G= A x B;
 C.dump("dot");

The supported operations are x (multiply), . (concatenate), not (complement), rev (reverse), int (intersection), and + (union)
